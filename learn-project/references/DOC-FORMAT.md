# 文档格式规范

## 阶段文档结构（01 ~ 06）

每个阶段文档遵循以下结构：

```markdown
# 标题

## 学习目标
（一句话说明这个阶段要搞懂什么）

## 正文内容
（结构化的分析内容，善用表格、流程图、代码引用）

## 关键文件索引
（本阶段涉及的关键源码文件路径列表）

## 自检问题
（3-5个问题，用于验证理解程度）
```

## 问答记录格式（07-qa-*.md）

```markdown
### Q: 用户的问题
A: 回答的要点摘要（不需要完整复制对话，提炼关键信息）
相关文件：`path/to/file.py`
```

## 写作规则

- 默认使用中文（除非用户要求其他语言）
- 引用源码时标注完整的相对文件路径，方便跳转
- 善用表格整理对比信息
- 流程图使用文本格式（ASCII art / 缩进箭头）
- 代码引用使用 markdown 代码块，标注语言
- 避免大段复制源码，提炼关键逻辑即可

## 文件命名规则

| 文件 | 内容 |
|------|------|
| `01-overview.md` | 项目概览 |
| `02-entrypoint.md` | 入口与启动流程 |
| `03-data-flow.md` | 核心数据流 |
| `04-modules.md` | 模块与服务拆解 |
| `05-deep-dive.md` | 关键实现深潜 |
| `06-getting-started.md` | 上手指南 |
| `07-qa-notes.md` | 问答笔记（通用） |
| `07-qa-{topic}.md` | 问答笔记（按主题拆分，超过5条时） |
