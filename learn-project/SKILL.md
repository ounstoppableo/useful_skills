# 通用项目学习技能

当用户想要系统性地学习一个陌生项目时，使用此技能引导分析并生成学习文档。

## 输出位置

所有学习产出文档存放在项目根目录的 `learning_cornerstone/` 文件夹中。

## 文件操作方式

所有文件的创建、追加、删除操作通过 shell 命令完成，不依赖特定 agent 的内置文件工具。

### 初始化输出目录

```bash
mkdir -p learning_cornerstone
```

### 写入文档

使用 heredoc 写入文件内容：

```bash
cat > learning_cornerstone/01-overview.md << 'DOCEOF'
（文档内容）
DOCEOF
```

### 追加内容（问答整理）

```bash
cat >> learning_cornerstone/07-qa-notes.md << 'QAEOF'

### Q: 用户的问题
A: 回答摘要
相关文件：`path/to/file`
QAEOF
```

### 清理重建

如果需要重新生成全部文档：

```bash
rm -rf learning_cornerstone
mkdir -p learning_cornerstone
```

### 删除单个文档

```bash
rm -f learning_cornerstone/05-deep-dive.md
```

## 执行模式

### 一键生成（初始学习）

当用户触发学习时，一次性完成全部 6 个阶段的分析，并将所有文档一次性生成到 `learning_cornerstone/`。不要逐阶段停下来等用户确认，直接全部输出。

执行顺序：
1. `mkdir -p learning_cornerstone`
2. 分析项目，生成 01 ~ 06 共 6 个文档
3. 创建 `07-qa-notes.md` 空模板

### 问答整理（持续学习）

文档生成后，用户会通过对话提问来深入学习。这些问答是宝贵的学习资料，需要持续整理：

1. 当用户提问并得到回答后，将问答内容按主题分类
2. 用 `cat >>` 追加到 `learning_cornerstone/07-qa-notes.md` 中
3. 如果某个主题的问答积累较多（5条以上），可以拆分为独立文件如 `07-qa-backend.md`、`07-qa-frontend.md` 等
4. 问答记录格式：

```markdown
### Q: 用户的问题
A: 回答的要点摘要（不需要完整复制对话，提炼关键信息）
相关文件：`path/to/file.py`
```

## 学习流程（6个阶段，一次性生成）

### 阶段1：项目概览 → `learning_cornerstone/01-overview.md`

目标：快速建立项目的第一印象。

执行步骤：
1. 阅读 README（如有）
2. 扫描项目根目录结构（package.json / pyproject.toml / Cargo.toml / go.mod 等）
3. 识别技术栈（语言、框架、数据库、外部服务）
4. 查看 Dockerfile / docker-compose（如有）
5. 查看 .env.example 或配置文件

输出内容：
- 项目一句话描述
- 技术栈清单
- 项目目录结构树（标注每个顶层目录的职责）
- 启动方式（开发/生产）
- 必要的环境变量和外部依赖

### 阶段2：入口与启动流程 → `learning_cornerstone/02-entrypoint.md`

目标：理解程序从启动到就绪的完整路径。

执行步骤：
1. 找到入口文件（main.py / index.ts / main.go / App.vue 等）
2. 追踪启动链：入口 → 配置加载 → 初始化 → 路由/中间件注册 → 监听端口
3. 如果是前后端分离项目，分别追踪两端的启动流程

输出内容：
- 入口文件路径
- 启动链流程图（文本格式）
- 配置加载机制
- 关键初始化步骤

### 阶段3：核心数据流 → `learning_cornerstone/03-data-flow.md`

目标：理解数据在系统中如何流转。

执行步骤：
1. 识别主要的用户操作/业务流程（3-5个核心流程）
2. 对每个流程追踪：用户操作 → API/路由 → 业务逻辑 → 数据存储 → 响应
3. 标注异步操作、消息队列、事件驱动等模式

输出内容：
- 核心业务流程列表
- 每个流程的数据流图（文本格式）
- 关键接口/API清单
- 数据存储方式

### 阶段4：模块与服务拆解 → `learning_cornerstone/04-modules.md`

目标：理解代码的组织方式和模块间的依赖关系。

执行步骤：
1. 识别分层架构（API层 / 服务层 / 数据层 / 工具层）
2. 列出每个模块/服务的职责
3. 分析模块间的调用关系
4. 识别核心设计模式（工厂、单例、观察者、中间件等）

输出内容：
- 分层架构图
- 模块职责清单（每个模块一句话描述）
- 模块依赖关系
- 使用的设计模式

### 阶段5：关键实现深潜 → `learning_cornerstone/05-deep-dive.md`

目标：深入理解项目中最复杂或最核心的实现。

执行步骤：
1. 识别项目中最复杂/最核心的 2-3 个模块
2. 阅读这些模块的源码
3. 分析关键算法、数据结构、设计决策
4. 记录非显而易见的实现细节

输出内容：
- 每个核心模块的详细分析
- 关键方法的调用链
- 重要的设计决策及其理由
- 需要注意的坑或特殊处理

### 阶段6：学习总结与上手指南 → `learning_cornerstone/06-getting-started.md`

目标：为后续开发提供快速参考。

输出内容：
- 开发环境搭建步骤
- 常见开发任务的操作路径（如：添加新API、添加新页面、修改业务逻辑）
- 调试技巧
- 自检问题清单（验证是否真正理解了项目）

## 执行规则

1. 一键生成：触发学习后，一次性完成全部6个阶段，不要中途停下等用户确认
2. 所有文件操作通过 shell 命令（mkdir / cat / rm）完成，不依赖 agent 内置文件工具
3. 如果项目规模较小，可以合并阶段
4. 如果项目有特殊领域知识（如模拟引擎、区块链、ML pipeline），在深潜阶段重点分析
5. 文档使用中文编写（除非用户要求其他语言）
6. 所有文档中引用源码时，标注文件路径，方便用户跳转查看
7. 每个文档末尾附上自检问题，帮助用户验证理解程度
8. 问答整理：用户后续提问的问答内容，用 `cat >>` 追加到 `learning_cornerstone/07-qa-notes.md`

## 文档格式规范

每个文档遵循以下结构：

```markdown
# 标题

## 学习目标
（一句话说明这个阶段要搞懂什么）

## 正文内容
（结构化的分析内容，善用表格、流程图、代码引用）

## 关键文件索引
（本阶段涉及的关键源码文件路径列表）

## 自检问题
（3-5个问题，用于验证理解程度）
```
